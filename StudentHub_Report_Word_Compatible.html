StudentHub - Collaborative Learning Platform
Technical Project Report

1. Executive Summary

StudentHub is a modern full-stack web application designed to facilitate collaboration among students through content sharing, discussion, and knowledge exchange. The platform serves as a centralized hub where students can share educational resources, post job opportunities, ask academic questions, and engage in meaningful discussions.

2. Technology Stack & Architecture

2.1 Backend Technologies

Framework: FastAPI (Python)

Database: MongoDB Atlas with Motor

Authentication: JWT Tokens (10-hour expiry)

File Storage: Cloudinary Cloud Service

Security: bcrypt hashing, Pydantic validation

CORS: FastAPI middleware configuration

2.2 Frontend Technologies

Framework: React 19.1.0

Build Tool: Vite 7.0.0

UI Library: React Bootstrap 2.10.10

Routing: React Router DOM 7.6.3

HTTP Client: Axios 1.10.0

Styling: Custom CSS + Bootstrap 5.3.7

3. System Architecture

The application follows RESTful API architecture with Single Page Application (SPA) frontend. Authentication uses JWT tokens with 10-hour expiration, file storage via Cloudinary cloud service, and MongoDB Atlas for data persistence.

3.1 Database Schema

Users Collection: user_id (UUID), username, email (unique), password (hashed), bio, created_at

Posts Collection: post_id (UUID), type (notes/jobs/queries), title, content, file_url, tags[], created_by, created_at

Comments Collection: comment_id (UUID), post_id (reference), content, created_by, created_at

6. Installation & Setup

6.1 Backend Setup

  1. Navigate to student_collabration directory
  2. Install dependencies: pip install -r requirements.txt
  3. Configure environment variables (.env file)
  4. Run server: python main.py (port 8080)

6.2 Frontend Setup

  1. Navigate to Student_hub_react directory
  2. Install dependencies: npm install
  3. Configure environment variables (.env file)
  4. Run development server: npm run dev (port 5173)

7. Frontend Component Architecture

7.1 Core Components

AuthForm: Handles user login/registration with form validation and error handling

Home: Main dashboard with post grid layout, search functionality, and filtering capabilities

Profile: User profile management, post history display, and account settings

Navigation: Responsive navigation bar with authentication state management

7.2 Utility Components

AuthGuard: Route protection component managing authentication state and access control

Loader: Animated loading component providing visual feedback during operations

PostCard: Reusable post display component with consistent styling and interactions

PostModal: Detailed post view with integrated comment system and file preview

8. Security Implementation

8.1 Authentication Security

JWT Security: 10-hour token expiration with HS256 algorithm

Password Security: bcrypt hashing with salt generation for secure storage

Route Protection: Middleware-based API endpoint security

Session Management: Secure token validation and refresh mechanisms

8.2 Data Protection

Input Validation: Pydantic models for comprehensive request validation

CORS Security: Properly configured cross-origin request handling

Injection Prevention: ODM usage prevents SQL injection attacks

File Security: Cloudinary secure upload handling with validation

9. File Management System

Cloudinary integration provides secure cloud-based file storage with automatic optimization, CDN delivery, and support for multiple file types including images, documents, and videos. Files are uploaded via multipart/form-data, processed with comprehensive error handling, and secure URLs are stored in the database for efficient retrieval.

11. Deployment & Scalability

Current development setup supports local backend (port 8080) and frontend (port 5173) environments. Production deployment options include Gunicorn/Uvicorn with Nginx reverse proxy for backend scalability, and static hosting platforms (Netlify/Vercel) for frontend distribution. MongoDB Atlas provides automatic database scaling, while Cloudinary handles file storage scaling requirements seamlessly.
