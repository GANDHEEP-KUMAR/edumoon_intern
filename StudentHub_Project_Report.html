StudentHub - Collaborative Learning Platform

Technical Project Report

Full-Stack Web Application for Educational Collaboration

Executive Summary

StudentHub is a modern, full-stack web application designed to facilitate collaboration among students through content sharing, discussion, and knowledge exchange. The platform serves as a centralized hub where students can share educational resources, post job opportunities, ask academic questions, and engage in meaningful discussions.

Technology Stack

Backend: FastAPI (Python), MongoDB Atlas, JWT Authentication, Cloudinary, bcrypt
Frontend: React 19.1.0, Vite 7.0.0, React Bootstrap, Axios, React Router DOM

Architecture

Pattern: RESTful API with SPA Frontend
Database: NoSQL (MongoDB)
Authentication: JWT-based
File Storage: Cloud-based (Cloudinary)

Core Features

  • Secure user authentication & registration
  • Content creation (Notes, Jobs, Queries)
  • File upload & sharing system
  • Interactive comment system
  • Advanced search & filtering
  • Responsive mobile-first design
  • Real-time content updates
  • User profile management
  • Tag-based content organization
  • Modern UI with animations

API Endpoints Overview

Method Endpoint Purpose Auth Required
POST /api/v1/user/sign-up User Registration No
POST /api/v1/user/login User Authentication No
POST /api/v1/post/create Create New Post Yes
GET /api/v1/post/all Retrieve All Posts No
GET /api/v1/post/by-user User's Posts Yes
POST /api/v1/comment/create Add Comment Yes

Database Schema

Users Collection:
• user_id (UUID)
• username, email (unique)
• password (hashed)
• bio (optional)

Posts Collection:
• post_id (UUID)
• type (notes/jobs/queries)
• title, content
• file_url, tags array
• created_by, created_at

Comments Collection:
• comment_id (UUID)
• post_id (reference)
• content
• created_by, created_at

Indexing Strategy:
• Users: email index
• Posts: created_by, type indexes
• Comments: post_id index

Frontend Component Architecture

Core Components:

  • AuthForm: Handles login/registration with validation
  • Home: Main dashboard with post grid, search, and filtering
  • Profile: User profile management and post history
  • Navigation: Responsive navbar with auth state

Utility Components:

  • AuthGuard: Route protection and authentication
  • Loader: Animated loading states
  • PostCard: Reusable post display component
  • PostModal: Detailed post view with comments

Security Implementation

Authentication Security:
• JWT tokens with 10-hour expiration
• bcrypt password hashing with salt
• Protected route middleware
• Session token validation

Data Protection:
• Pydantic model validation
• CORS configuration
• SQL injection prevention
• Secure file upload handling

File Management System

Cloudinary Integration: The application uses Cloudinary for secure file storage and delivery. Files are uploaded via multipart/form-data, processed by Cloudinary, and secure URLs are stored in the database. The system supports images, documents, and videos with automatic optimization and CDN delivery.

Deployment & Scalability

Current Setup: The application is configured for development with local backend (port 8080) and frontend (port 5173). For production deployment, the backend can be deployed using Gunicorn/Uvicorn with Nginx reverse proxy, while the frontend can be deployed to static hosting services like Netlify or Vercel.

Scalability Considerations: The MongoDB Atlas database provides automatic scaling, Cloudinary handles file storage scaling, and the modular architecture supports horizontal scaling. Future enhancements could include Redis caching, load balancing, and microservices architecture.
