StudentHub - Collaborative Learning Platform
Project Report

1. Project Overview

StudentHub is a full-stack web application designed to facilitate collaboration among students through content sharing, discussion, and knowledge exchange. The platform enables students to share educational resources, post job opportunities, ask academic questions, and engage in meaningful discussions through an intuitive interface.

1.1 Key Features

  • User authentication and registration system
  • Content creation and management (Notes, Jobs, Queries)
  • File upload and sharing capabilities
  • Interactive comment system for discussions
  • Advanced search and filtering functionality
  • Responsive design for mobile and desktop
  • User profile management
  • Tag-based content organization

1.2 Project Metrics

  • 15+ React Components
  • 8 API Endpoints
  • 3 Content Types (Notes, Jobs, Queries)
  • 3 Database Collections
  • 100% Responsive Design Implementation

2. Technology Stack

2.1 Backend Technologies

  • Framework: FastAPI (Python)
  • Database: MongoDB Atlas with Motor async driver
  • Authentication: JWT (JSON Web Tokens)
  • File Storage: Cloudinary
  • Password Security: bcrypt hashing
  • Validation: Pydantic models
  • CORS: FastAPI CORS middleware

2.2 Frontend Technologies

  • Framework: React 19.1.0
  • Build Tool: Vite 7.0.0
  • UI Library: React Bootstrap 2.10.10
  • Routing: React Router DOM 7.6.3
  • HTTP Client: Axios 1.10.0
  • Styling: Custom CSS with Bootstrap 5.3.7

3. System Architecture

The application follows a RESTful API architecture with a Single Page Application (SPA) frontend. The backend provides API endpoints for data operations, while the frontend handles user interface and interactions. Authentication is managed through JWT tokens, and file storage is handled by Cloudinary cloud service.

3.1 Database Schema

Users Collection: user_id (UUID), username, email (unique), password (hashed), bio (optional), created_at timestamp

Posts Collection: post_id (UUID), type (notes/jobs/queries), title, content, file_url, tags array, created_by (email), created_at timestamp

Comments Collection: comment_id (UUID), post_id (reference), content, created_by (email), created_at timestamp

4. API Endpoints

Method Endpoint Purpose Authentication
POST /api/v1/user/sign-up User Registration No
POST /api/v1/user/login User Authentication No
POST /api/v1/post/create Create New Post Yes
GET /api/v1/post/all Retrieve All Posts No
GET /api/v1/post/by-user Get User's Posts Yes
GET /api/v1/post/by-post/{post_id} Get Post Comments No
POST /api/v1/comment/create Add Comment Yes

5. File Structure

Backend (student_collabration/): main.py (entry point), ROUTES/ (API handlers), data.py (Pydantic models), utils.py (utility functions), middlieware.py (authentication), DBcreation.py (database connection), cloudinary_utils.py (file upload)

Frontend (Student_hub_react/): src/App.jsx (main component), src/components/ (React components), src/main.jsx (entry point), public/ (static assets)

6. Frontend Component Architecture

6.1 Core Components

  • AuthForm: Handles user login and registration with form validation and error handling
  • Home: Main dashboard displaying posts with search, filtering, and post creation functionality
  • Profile: User profile management, post history, and account settings
  • Navigation: Responsive navigation bar with authentication state management

6.2 Utility Components

  • AuthGuard: Route protection component managing authentication state
  • Loader: Animated loading component for better user experience
  • PostCard: Reusable component for displaying post information
  • PostModal: Detailed post view with comment system integration

7. Security Implementation

7.1 Authentication Security

  • JWT tokens with 10-hour expiration time
  • bcrypt password hashing with salt generation
  • Protected route middleware for API endpoints
  • Session token validation on protected routes
  • Secure password storage (only hashed passwords in database)

7.2 Data Protection

  • Pydantic model validation for all API requests
  • CORS configuration for secure cross-origin requests
  • SQL injection prevention through ODM usage
  • Secure file upload handling via Cloudinary
  • Input sanitization and type checking

8. File Management System

The application uses Cloudinary for secure cloud-based file storage. Files are uploaded via multipart/form-data, processed by Cloudinary with automatic optimization, and secure URLs are stored in the database. The system supports images, documents, and videos with CDN delivery for optimal performance.

9. Installation and Setup

9.1 Backend Setup

  1. Navigate to student_collabration directory
  2. Install dependencies: pip install -r requirements.txt
  3. Configure environment variables (MongoDB URI, Cloudinary credentials, JWT secret)
  4. Run server: python main.py (starts on port 8080)

9.2 Frontend Setup

  1. Navigate to Student_hub_react directory
  2. Install dependencies: npm install
  3. Configure environment variables (backend URL)
  4. Run development server: npm run dev (starts on port 5173)

11. Deployment and Scalability

The current setup supports development with local backend (port 8080) and frontend (port 5173). For production deployment, the backend can be deployed using Gunicorn/Uvicorn with Nginx reverse proxy, while the frontend can be hosted on static hosting services like Netlify or Vercel. The MongoDB Atlas database provides automatic scaling capabilities, and Cloudinary handles file storage scaling requirements.

12. Conclusion

StudentHub successfully demonstrates modern full-stack development practices with FastAPI and React. The project implements secure authentication, efficient file handling, responsive design, and comprehensive API functionality. The modular architecture ensures maintainability and provides excellent scalability potential for future educational collaboration features. The application serves as a solid foundation for a production-ready educational platform with clear enhancement pathways.
