StudentHub - Collaborative Learning Platform
Project Report

1. Project Overview

StudentHub is a full-stack web application designed to facilitate collaboration among students through content sharing, discussion, and knowledge exchange. The platform enables students to share educational resources, post job opportunities, ask academic questions, and engage in meaningful discussions.

1.1 Key Features & Metrics

  • User authentication and registration
  • Content creation (Notes, Jobs, Queries)
  • File upload and sharing capabilities
  • Interactive comment system
  • Advanced search and filtering
  • Responsive design for all devices
  • 15+ React Components
  • 8 API Endpoints
  • 3 Content Types
  • 3 Database Collections
  • 100% Responsive Design
  • JWT Authentication

2. Technology Stack

2.1 Backend Technologies

Framework: FastAPI (Python)

Database: MongoDB Atlas with Motor

Authentication: JWT Tokens

File Storage: Cloudinary

Security: bcrypt, Pydantic validation

CORS: FastAPI middleware

2.2 Frontend Technologies

Framework: React 19.1.0

Build Tool: Vite 7.0.0

UI Library: React Bootstrap 2.10.10

Routing: React Router DOM 7.6.3

HTTP Client: Axios 1.10.0

Styling: Custom CSS + Bootstrap 5.3.7

3. System Architecture & Database Schema

The application follows RESTful API architecture with SPA frontend. Authentication uses JWT tokens, file storage via Cloudinary cloud service.

3.1 Database Collections

Users: user_id (UUID), username, email (unique), password (hashed), bio, created_at

Posts: post_id (UUID), type (notes/jobs/queries), title, content, file_url, tags[], created_by, created_at

Comments: comment_id (UUID), post_id (reference), content, created_by, created_at

6. Installation & Setup

6.1 Backend Setup

  1. cd student_collabration
  2. pip install -r requirements.txt
  3. Configure environment variables
  4. python main.py (port 8080)

6.2 Frontend Setup

  1. cd Student_hub_react
  2. npm install
  3. Configure .env file
  4. npm run dev (port 5173)

7. Frontend Component Architecture

8. Security Implementation

8.1 Authentication & Data Protection

JWT Security: 10-hour token expiration, HS256 algorithm

Password Security: bcrypt hashing with salt generation

Route Protection: Middleware for API endpoint security

Session Management: Secure token validation

Data Validation: Pydantic models for request validation

CORS Security: Configured cross-origin requests

Injection Prevention: ODM usage prevents SQL injection

File Security: Cloudinary secure upload handling

9. File Management System

Cloudinary integration provides secure cloud-based file storage with automatic optimization, CDN delivery, and support for images, documents, and videos. Files are uploaded via multipart/form-data, processed with error handling, and secure URLs stored in database.

11. Deployment & Scalability

Current development setup supports local backend (port 8080) and frontend (port 5173). Production deployment options include Gunicorn/Uvicorn with Nginx reverse proxy for backend, and static hosting (Netlify/Vercel) for frontend. MongoDB Atlas provides automatic scaling, while Cloudinary handles file storage scaling requirements.
